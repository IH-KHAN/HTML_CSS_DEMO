14.1 Box Sizing Types (Layout Calculation)

The `box-sizing` property defines how the width and height of an element are calculated: should they include padding and borders, or not? This is critical for pixel-perfect financial reports.

14.1.1 Border Box (Recommended)

Width: 300px (Fixed). Padding and Border are INCLUDED in this width.

This ensures the element stays exactly 300px wide, fitting perfectly into grid layouts.


14.1.2 Content Box (Default)

Width: 300px + 40px Padding + 20px Border = 360px Total.

The element grows larger than 300px because padding/border are added *outside* the width.

14.2 CSS Margins (External Spacing)

Margins create space *outside* the border, separating this component from others.

14.2.1 Auto Margin (Centering)

This report card is centered using `margin: 0 auto;`.


14.2.2 Collapsing Margins

Paragraph 1 (Margin Bottom: 30px)
Paragraph 2 (Margin Top: 20px)
Result: The distance between these two is **30px** (the larger of the two), not 50px.

14.3 CSS Outline (Focus Indicators)

An outline is a line drawn *outside* the elements border. Unlike borders, outlines do not take up space. They are often used to highlight active input fields for accessibility.

Active Transaction Input Field (Highlighted with Green Outline)

14.4 CSS Borders (Component Boundaries)

14.4.1 Border Styles

Solid: Standard Data Card
Dashed: Pending Transaction
Dotted: Optional Field
Double: Total Summary Line

14.4.2 CSS Border Radius (Modern UI)

Rounded corners (`border-radius: 15px`) create a softer, modern look for client-facing dashboards.


14.4.3 CSS Border Images

(Advanced feature, rarely used in strict banking UIs, but useful for marketing banners).
border-image: url(border.png) 30 round;

14.5 CSS Padding (Internal Spacing)

Padding creates space *inside* the border, between the border and the content. It ensures text doesn't touch the edges of the card.
