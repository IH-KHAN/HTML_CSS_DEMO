22.1 Responsive Web Design (RWD)

RWD ensures that our banking application looks good and functions correctly on all devices, from smartphones to large desktop monitors. It uses HTML and CSS to resize, hide, shrink, enlarge, or move content to make it look good on any screen.

[Image of Responsive Web Design Layouts]

22.2 RWD Viewport

The viewport is the user's visible area of a web page. It varies with the device. To ensure proper scaling, we must include the following `` tag in all our HTML headers: 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Without this, mobile browsers will render the page at desktop width and shrink it down, making text unreadable.

22.3 CSS Media (Queries & Types)

22.3.1 Media Types

Common types include:

  • all: Used for all media type devices.
  • print: Used for printers (e.g., printing a bank statement).
  • screen: Used for computer screens, tablets, smart-phones.

22.5 Mobile-First vs. Desktop-First

22.5.1 Mobile-First Strategy (Recommended)

We design for the smallest screen first (default styles), then add complexity for larger screens using `min-width` media queries. This is more performant for mobile devices.

22.5.2 Desktop-First Strategy

Designing for large screens first, then using `max-width` queries to override styles for smaller screens. This is less common in modern web development but useful for legacy banking desktop apps. 

/* Desktop styles first */
.container { width: 1000px; }

/* Mobile Override */
@media (max-width: 768px) {
  .container { width: 100%; }
}
