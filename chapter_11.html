11.1 Syntax & 11.2 Comments (System Documentation)

CSS (**Cascading Style Sheets**) governs the visual presentation of our financial dashboard. Correct syntax is critical for consistent application across all browsers.

11.1 CSS Syntax

A rule set consists of a selector, property, and value, defining how a component looks.

            /* Selector targets the element to be styled */
            .account-balance {
              /* Property: Value; - Declaration Block */
              color: #004d99; /* Declares the text color */
              font-size: 1.5rem; /* Declares the text size */
            }

11.2 Comments in CSS

Comments are used for documenting complex styles, audit trails, and separating style sections. They are denoted by `/* comment */`. 

            /* --- BEGIN Q4 2025 STYLING AUDIT --- */
            /* Ensure all critical alerts use 'red' */
            .alert-critical {
              /* Do not delete: Required by Accessibility Standard AA */
              color: red;
            }

11.3 CSS Units (Precision and Scaling)

Choosing the correct unit dictates whether a component's size is **fixed** (for print) or **responsive** (for dashboards).

11.3.1 Absolute Lengths (Fixed Dimensions)

Absolute units (like **px** and **pt**) are fixed and do not scale with the screen size. They are suitable for elements that must maintain a specific physical dimension.

Pixel (px) Example: Fixed Icon Size

This text size is 24px.

Point (pt) Example: Print Typography

This text size is 18pt.

11.3.2 Relative Lengths (Adaptive Scaling)

Relative units scale based on a reference point (e.g., parent font size or viewport size), making them essential for a **responsive financial dashboard**.

11.3.2.1 em (Relative to Parent's Font Size)

Parent Font Size: 20px

Child text is 1.5em (scales based on this parent box).

11.3.2.2 rem (Relative to Root Font Size)

This text is 1.5rem (scales based on the browser's default setting, typically 16px).


11.4 Shorthand Properties (Coding Efficiency)

Shorthand properties combine multiple related CSS declarations into a single, efficient line. This reduces file size and complexity, improving system performance.

Shorthand Example: Margin, Padding, and Border

Shorthand Border/Margin/Padding (Efficient Code) 

border: 5px solid #00a04d; margin: 20px 0 10px 0; padding: 10px 15px 10px 15px;

Non-Shorthand Comparison (Verbose Code)

width: 300px;
              height: 100px;
              border-width: 5px;
              border-style: solid;
              border-color: #cc0000;
              margin-top: 20px;
              margin-bottom: 10px;
              margin-left: 0;
              margin-right: 0;
              padding-top: 10px;
              padding-bottom: 10px;
              padding-left: 15px;
              padding-right: 15px;
              background-color: #ffffff;

11.5 CSS Placement & 11.6 User Agent Styles

11.5 Where Do We Put Our CSS? (Deployment Strategy)

The placement of CSS affects its scope and priority (cascading).

11.5.1 Inline CSS (Local Overrides)

Inline CSS is applied directly to an element using the `style` attribute (e.g., this text is red). Use sparingly for local, one-off overrides.

11.5.2 Internal CSS (Page-Specific Styles)

Internal CSS is placed within the <style> tags in the document's <head>. It is used for styles unique to this single page. (This box is styled via Internal CSS).

11.5.3 External CSS (Global Branding)

External CSS is in a separate file (e.g., `banking_style.css`) and linked via <link>. This is the **standard method** for global branding and maintaining consistency across the entire banking platform.

11.6 User Agent Style Sheets

These are the default styles provided by the browser (e.g., Chrome, Firefox). They define how a default `<h1>` looks. Your custom CSS **overrides** these default User Agent styles.
