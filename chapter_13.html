13.1 Rule 1: The Last Declaration Will Win

If two selectors are identical, the browser uses the one defined last in the CSS file. This represents the most recent instruction.

System Status: Offline (This text is RED because the red rule was declared last).

13.2 Rule 2: The More Specific Declaration Will Win

Browsers calculate a "specificity score" for every rule. A rule with a higher score overrides a lower one, regardless of order.
Hierarchy: ID (#id) > Class (.class) > Element (tag).

Standard Client (Styled by Class .account-info) - Blue

Premium Client (Styled by ID #premium-client) - Green [ID beats Class]

13.3 Rule 3: Inline Style is Preferred

Styles applied directly to an HTML element via the `style="..."` attribute override both Internal (in <head>) and External (in .css file) styles.

High Priority Notice (Orange) - Inline style overrides the Blue class style.

13.4 Rule 4: “!important” is the Strongest Rule

The `!important` declaration overrides all other rules, including inline styles. It acts as a mandatory compliance enforcement.

MANDATORY AUDIT REQUIRED. (Red) - The `!important` rule forces this to be red, ignoring the inline black style.

13.5 Inheritance

Some CSS properties (like `font-family` and `color`) are passed down from parent elements to their children. This ensures consistency across a data card without restating rules.
