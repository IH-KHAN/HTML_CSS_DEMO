1.1 Two Types of HTML Elements

In banking applications, we use two main types of elements to structure data:

1. Container Elements (Normal Elements)

These have a start tag and an end tag. They contain content like account balances or security notices.

<h1>Welcome to SecureBank</h1>
<p>Your session is secure.</p>

2. Void Elements (Empty Elements)

These have only a start tag and no content. Used for visual breaks or metadata.

<hr> (Thematic Break - Separating sections)
<br> (Line Break - Formatting addresses)
<img src="logo.png"> (Images)

1.2 HTML Document Structure

Every compliant financial webpage follows a strict tree structure (DOM). This ensures security tools and screen readers can parse the data correctly.

<!DOCTYPE html> <html> <head> <!-- Metadata: Title, Charset, Security Policies --> </head> <body> <!-- Visible Content: Dashboard, Accounts, Forms --> </body> </html>

1.4 Character Encoding (Global Banking)

Banking is global. We must use UTF-8 to ensure currencies (¥, €, £) and languages (Bengali, Japanese, Arabic) display correctly without data corruption.

<meta charset="UTF-8" />

Examples of Multilingual Support:

  • English: Welcome to My Webpage!
  • Bengali: আমার ওয়েবপেজে স্বাগতম! (Welcome to my webpage!)
  • Japanese: 私のウェブページへようこそ！ (Welcome to my webpage!)
  • Arabic: مرحبًا بكم في صفحتي الإلكترونية! (Welcome to my electronic page!)

1.5 Element Attributes (Configuration)

Attributes provide extra information about elements. In banking, they are crucial for linking stylesheets, setting input types, and accessibility.

<input type="password" placeholder="Enter Pin" />
  • Name: The attribute name (e.g., `type`).
  • Value: The setting (e.g., `password`).

1.6 Favorite Icon (Favicon)

The Favicon is the small logo that appears in the browser tab. It is essential for brand recognition and phishing protection (users look for the bank logo).

<link rel="icon" href="icon/favicon.png" />

(This icon appears in your tab)
